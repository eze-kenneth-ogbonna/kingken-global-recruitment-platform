## Field Detail

### Column A тАФ CandidateID
- **Format:** `KENG-YYYYMMDD-NNNN` (e.g. `KENG-20260115-0042`)
- **Generation:** Apps Script generates on form submission using today's date + sequential counter
- **Uniqueness:** Script checks existing IDs before assigning; duplicate prevention enforced
- **Read-only:** Protected range тАФ only the Apps Script service account writes this field

### Column B тАФ Timestamp
- **Source:** Automatically copied from Google Form `Timestamp` column in Form Responses
- **Format:** `YYYY-MM-DD HH:MM:SS` (24-hour)
- **No manual edits** тАФ protected range

### Column C тАФ FullName
- **Required:** Yes
- **Validation:** Minimum 3 characters; no purely numeric values
- **Casing:** Title Case preferred; normalize via Apps Script

### Column D тАФ Phone
- **Format:** E.164 тАФ starts with `+`, country code, then number (no spaces, dashes, parentheses)
- **Example valid values:** `+23480123456789`, `+96512345678`
- **Normalization script:** Strips spaces and non-numeric except leading `+`
- **Uniqueness check:** Apps Script flags duplicates in column P (Notes)

### Column E тАФ Country
- **Accepted values:** Nigeria, Ghana, Kenya, Ethiopia, Uganda, Tanzania, Cameroon, Senegal, Other
- **Source:** From form dropdown; may be manually updated

### Column F тАФ Position
- **Accepted values:** Domestic Worker, Driver, Nanny/Caregiver, Cook/Chef, Cleaner, Security Guard, Construction Worker, Factory Worker, Waiter/Waitress, Receptionist, Other
- **Source:** From form dropdown

### Column G тАФ ExperienceYears
- **Type:** Integer (whole number)
- **Range:** 0тАУ50
- **Data validation:** Must be a number тЙе 0

### Column H тАФ PassportAvailable
- **Accepted values:** `Yes - valid`, `Yes - but expired`, `No - applying`, `No - need assistance`
- **Used for:** Filtering candidates ready for immediate processing

### Column I тАФ CV_Link
- **Format:** Full URL starting with `https://`
- **Source:** Google Form file upload тЖТ auto-uploaded to Google Drive тЖТ link stored here
- **Optional field** тАФ may be blank; recruiter can manually paste link later

### Column J тАФ SourceSheetRow
- **Purpose:** Traceability link back to Form Responses sheet row
- **Auto-set** by Apps Script on import

### Column K тАФ Status (Main Tracker)
- **Type:** Dropdown (Data Validation)
- **Allowed values:** `New`, `Screened`, `Approved`, `Rejected`, `Processing`, `Deployed`
- **Workflow progression:**
  ```
  New тЖТ Screened тЖТ Approved тЖТ Processing тЖТ Deployed
                тЖШ Rejected (at any stage)
  ```
- **Stage descriptions:**
  - `New` тАФ Just submitted, not yet reviewed
  - `Screened` тАФ AI score assigned + recruiter has reviewed
  - `Approved` тАФ Recruiter approved, ready for employer matching
  - `Rejected` тАФ Not suitable; reason in Notes column
  - `Processing` тАФ Active placement in progress (visa, documents)
  - `Deployed` тАФ Successfully placed and deployed to employer

### Column L тАФ AI_Score
- **Range:** 0тАУ100 (integer)
- **Source:** OpenAI API via Zapier automation; score based on experience, position match, document readiness
- **Protected range:** Read-only for Recruiter role; editable only by Tech Lead
- **Scoring guide:**
  - 80тАУ100 тЖТ Strong candidate, prioritize
  - 60тАУ79 тЖТ Good candidate, review further
  - 0тАУ59 тЖТ Weak match, likely reject unless manual override

### Column M тАФ AssignedTo
- **Values:** Team member name (from WATI/Notion team roster)
- **Default:** Assigned by COO or auto-assigned by Zapier based on load balancing

### Column N тАФ ScreeningDate
- **Format:** `YYYY-MM-DD`
- **Auto-set** when Status changes from New тЖТ Screened (Apps Script trigger)

### Column O тАФ VisaStatus
- **Accepted values:** `Not Started`, `Applied`, `Approved`, `Issued`
- **Used for:** Visa processing pipeline tracking

### Column P тАФ Notes
- **Type:** Free text; unlimited length in Sheets (no hard cap)
- **Usage:** Recruiter observations, duplicate flags, special requirements

### Column Q тАФ LastUpdated
- **Format:** `YYYY-MM-DD HH:MM:SS`
- **Auto-updated** by Apps Script `onEdit` trigger whenever any column AтАУP is modified
- **Protected from manual edit**

## Data Validation Rules

```
Column K (Status):
  Type: List from range (or custom list)
  Values: New,Screened,Approved,Rejected,Processing,Deployed
  Show dropdown in cell: Yes
  Reject invalid input: Yes

Column G (ExperienceYears):
  Type: Number
  Criteria: Is a valid number, between 0 and 50
  Reject invalid input: Yes

Column I (CV_Link):
  Type: Text
  Criteria: URL begins with https://
  Show warning on invalid: Yes (not hard reject тАФ URL may be blank)
```

**Setting up protection in Google Sheets:**
1. Select the column(s) to protect
2. Right-click тЖТ Protect range
3. Click "Set permissions"
4. Select "Only you" or add specific editors
5. Save

## Key Formulas

### Auto-populate ScreeningDate when Status changes to "Screened"
*(Handled by Apps Script, not a sheet formula)*

### Count candidates by status (used in Dashboard)
```
=COUNTIF(K2:K,"New")
=COUNTIF(K2:K,"Approved")
=COUNTIF(K2:K,"Deployed")
```

### Average AI Score of Approved candidates
```
=AVERAGEIF(K2:K,"Approved",L2:L)
```

### Find candidate by phone number
```
=MATCH("+23480123456789",D2:D,0)
```

### Check for duplicate phone numbers
```
=COUNTIF(D:D,D2)>1
```
*(Apply as conditional formatting rule to column D тАФ highlight if >1)*
