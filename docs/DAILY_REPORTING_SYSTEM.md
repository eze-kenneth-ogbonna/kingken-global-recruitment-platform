## Overview

The daily reporting system ensures every team member stays accountable, the COO has full visibility into operations, and the business maintains momentum every day. Reports are submitted via WhatsApp and reviewed by the COO.

## 2. Submission Rules

| Rule | Detail |
|------|--------|
| Submission deadline | 6:00 PM every working day (Monday–Saturday) |
| Submission channel | Operations Daily WhatsApp group |
| Format | Use exact template above — no abbreviations |
| Minimum detail | Each section must have at least one entry (use "None today" if no activity) |
| Sunday | No report required unless a specific task was assigned |
| Public holidays | Report still required if you worked; optional if official holiday |
| Remote/travel | Report required regardless of location |

## 4. Accountability System

### Compliance Tracking

The COO tracks report submissions in a **Report Log** column in the Tasks tab or a separate tracking sheet.

| Violation Level | Definition | Consequence |
|----------------|-----------|-------------|
| Late submission | Report received after 6 PM | Verbal reminder via WhatsApp |
| First missed report | No report submitted for a working day | Entry recorded; private message from COO |
| Second missed report | Second no-show within same month | Written notice sent via WhatsApp (documented) |
| Third missed report | Third no-show in same month | Formal written warning issued by COO |
| Continued non-compliance | 4th+ miss | Performance review meeting with COO and CEO |

### COO Review Process

Every evening, the COO performs a **15-minute review**:

```
Step 1 (2 min): Count number of reports received
Step 2 (5 min): Read each report quickly — flag any blockers
Step 3 (3 min): Identify any missed reports — send private reminder
Step 4 (5 min): Update COO's own summary for CEO
```

## 6. Weekly Team Meeting Template

**Frequency:** Every Monday, 10:00 AM WAT
**Format:** WhatsApp group call or Google Meet
**Duration:** 30–45 minutes
**Facilitator:** COO
**Note-taker:** HR/Admin

```
WEEKLY TEAM MEETING AGENDA
Week of: [Date Range e.g., 15–19 Jan 2024]

1. LAST WEEK REVIEW (10 min)
   - Total candidates processed: [number]
   - Total deployments: [number]
   - Revenue collected (USD): [amount]
   - KPIs hit / missed (refer to Dashboard)

2. THIS WEEK PRIORITIES (10 min)
   - Top 3 candidates to process
   - Top 3 employers to follow up
   - Any urgent deals or payments due

3. BLOCKERS AND SUPPORT NEEDED (10 min)
   - Each team member shares one blocker (if any)
   - COO assigns support or action

4. ANNOUNCEMENTS (5 min)
   - Any new policies, system changes, or platform updates
   - Company news from CEO

5. CLOSING (2 min)
   - Action items confirmed (who does what by when)
   - Next meeting confirmed
```

### Weekly Meeting Minutes Format

After each meeting, HR/Admin posts minutes in Operations Daily group:

```

• [Name] to [action] by [date]

• Candidates: [x] | Deployments: [x] | Revenue: $[x]

Next Meeting: [Date/Time]
```

## 8. Escalation Protocol

When a daily report contains a blocker or critical issue, the following escalation path applies:

| Issue Type | Who to Inform | Response Time |
|-----------|--------------|---------------|
| System outage (Zap/Bot down) | Head of Tech immediately | Within 1 hour |
| Candidate in danger / abuse complaint | COO + Head of Legal immediately | Within 30 minutes |
| Employer dispute or non-payment | COO immediately | Within 2 hours |
| Team performance concern | COO → private conversation | Within 24 hours |
| Platform data breach | Head of Tech + COO + CEO | Within 15 minutes |

Use this message format for urgent escalation in the Operations Daily group:

```
🚨 URGENT ESCALATION — [Type]
From: [Your Name]
Issue: [Brief description]
Impact: [What is affected]
Action needed: [What you need]
Urgency: [CRITICAL / HIGH / MEDIUM]
Time: [Current time]
```
